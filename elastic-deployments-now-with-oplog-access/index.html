Elastic Deployments, now with oplog access

March 20th, 2014

We’ve just released a feature that allows all customers on our Elastic DeploymentsMongoDB’s oplog plays an important role in keeping a MongoDB replica set in sync with the primary member’s state (our source of truth). As the ecosystem around MongoDB has matured, we’ve seen a proliferation of tools taking advantage of the oplog for really clever technical features.

The real-time updating web framework Meteor is using oplog data in its latest releases to synchronize changing data in MongoDB with connected clients without having to poll the MongoDB server for better scale and performance.

Elasticsearch can also take advantage of the oplog to keep a document index in sync by using MongoDB River to react to changes in the Mongo database. There are also a range of lower level libraries and utilities which can be customized to allow your applications to react in real-time to changes in the database.

Customers with existing Elastic Deployments can access the feature now. If you aren’t yet live MongoHQ, sign up now for immediate access.

More technical information about the feature is available on our documentation site. If you have any questions please get in touch.
